# /alert - 监控告警管理

管理股票监控服务，支持启动、停止监控，查看历史告警记录。

## 使用方式

```
/alert <command> [options]
```

## 命令

| 命令 | 说明 | 示例 |
|------|------|------|
| `start` | 启动监控服务 | `/alert start` |
| `stop` | 停止监控服务 | `/alert stop` |
| `status` | 查看监控状态 | `/alert status` |
| `history` | 查看历史告警 | `/alert history` |
| `history <code>` | 查看指定股票历史告警 | `/alert history 000001` |

## 提醒渠道

监控触发时可通过以下渠道发送告警：

- `terminal` - 终端输出（默认）
- `notify` - 系统通知

通过 `/watch add` 命令设置提醒渠道。

## 工作原理

1. 监控服务启动后，定期扫描监控列表中的股票
2. 使用 SignalScanner 检测技术信号
3. 检测到信号后，通过 AlertEngine 发送告警
4. 告警记录保存到数据库

## 注意事项

- 监控服务仅在交易时间运行（9:30-11:30, 13:00-15:00）
- 建议在开盘前启动监控服务
- 可通过配置文件调整扫描间隔
